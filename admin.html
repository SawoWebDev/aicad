OpenRouter & models
Used for chat + image generation.
Create one at openrouter.ai → Management Keys. Used only to fetch your credit balance.
Pipeline 2 uses the cheap Conversation model for the back-and-forth requirements chat, then switches to the Chat model (keep this on Opus) to finalize the dataset and image prompt, then renders with the Image model. This avoids spending Opus tokens on basic Q&A.
Which conversation model should I pick?
ModelCostFollows format rulesBest for
Claude Haiku 4.5 recommended $ Excellent Default. Same family as Opus, so it reliably obeys the plain-text / no-em-dash / trigger-tag rules. Big savings, near-zero risk.
Gemini 2.5 Flash Lite $ Good Fastest option (~400ms, ~88 t/s) and very cheap. Best UX of the budget picks. Occasionally slips on formatting; the widget strips most of it.
Gemini 2.5 Flash $ Good Cheap and fast. Occasionally slips on formatting (stray asterisks / dashes) — the widget strips most of that anyway.
DeepSeek V4 Flash $ Unverified Cheapest per token, but slower to reply (~15 t/s) and not from the Claude family — test the trigger-tag handoff before relying on it. If it drops the tag, the chat won't advance to the drawing.
Tencent Hy3 preview $ Unverified Cheapest input cost, but ~4.5s to first token. Same caveat: verify it emits the trigger tag and obeys the plain-text rules before using in production.
Claude Sonnet 4.6 $$ Excellent Smarter small-talk and edge-case handling than Haiku, still well under Opus. Use if Haiku ever feels too literal.
GPT-5.4 $$ Good Strong reasoning; only pick if you specifically prefer OpenAI for the chat.
The conversation model only handles Q&A. The Chat model below (keep on Opus) still does the precise finalize + image-prompt step, so drawing quality is unaffected by this choice.
Labels are a rough guide: $ = cheap/testing, $$$ = most capable/expensive. All IDs are OpenRouter slugs.
